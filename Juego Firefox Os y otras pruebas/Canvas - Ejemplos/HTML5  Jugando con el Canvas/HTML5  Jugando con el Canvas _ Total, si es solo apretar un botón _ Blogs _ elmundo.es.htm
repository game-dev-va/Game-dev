HTML5: Jugando con el Canvas

30 MAR 2011 11:31

Por Ricardo Moreno

Después de teorizar en dos posts anteriores, toca 'darle a la tecla'. Es el momento de un experimento rápido y divertido que puede dar alguna idea para posteriores desarrollos en HTML5.

Hoy voy a hablar del elemento Canvas y voy a realizar un ejemplo paso a paso de cómo representar y animar un 'sprite' de un videojuego. El ejemplo es relativamente sencillo y no está depurado al 100%, pero puede ser un buen comienzo para desarrollar temas más complejos de los que, evidentemente, quiero un porcentaje.

Antes de nada, quiero dejar claro el resultado final. Cuando todo acabe deberá verse como en este enlace. Y con las flechas del cursor podremos hacer que el muñeco ande hacia adelante y hacia atrás.

Ahora, un par de líneas de historia

El elemento Canvas lo introdujo en el HTML, como muchas otras cosas, Apple. Después de ser 'implementado' en Safari y Chrome, 'los Gecko' comenzaron a incluir este elemento. Actualmente es un elemento incluido dentro de HTML5 y que puede ser utilizado -y traduzco de la documentación 'The Markup Language Reference'- para representar "gráficos" de forma dinámica -y por lo tanto, nos vale para nuestro propósito-.

¿Qué necesitamos?

Como vamos a hacer una animación de un 'sprite', necesitamos los 'frames' que representan las posiciones del 'sprite' a animar. Aquí pordemos ver un ejemplo de la hoja completa de, en este caso, una tanqueta del famoso juego Metal Slug (gracias a Javier del Moral por la referencia). En nuestro caso, os dejo este enlace con los siete 'frames' de nuestro personaje que, evidentemente, se utilizará exclusivamente con fines pedagógicos para este post.

Primeros pasos

Lo que haremos será crear una animación del personaje. Esta animación será representada en un Canvas y será lanzada cuando el usuario pulse 'cursor derecha' o 'cursor izquierda'. Evidentemente, también desplazaremos el 'sprite' en la coordenada 'X' y así daremos la sensación de movimiento. A su vez, comprobaremos que no se salga de los límites horizontales.

Para ello comenzamos creando el código HTML que alojará nuestro ejemplo.

<!DOCTYPE html>
<html>
<head>
  <meta charset="iso-8859-15" />
  <title>Jugando con el Canvas</title>
  <script src="js/canvas_animacion.js"></script>
  <style type="text/css">
    canvas { position: relative; }
    .precarga { position: absolute; left: -9999px; }
  </style> 
</head>

<body onload="init()">
  <div id="wrapper"></div>
</body>

</html>

Como se puede ver, es muy sencillo. Únicamente incluyo el fichero javascript que contendrá todas las funciones para nuestro fin; después creo un div con identificador 'wrapper' para crear el Canvas posteriormente y llamo a la función 'init' cuando se carga la página.

¡Vamos al lio!

1. Creo el elemento canvas desde javascript:

var canvas = document.createElement('canvas');

2. Recupero el contexto, que podemos definir como el lugar de operaciones. Es decir, donde vamos a disponer de un objeto en el que están definidas todas las propiedades y funciones de 'pintado', transformaciones, etc:

var ctx = canvas.getContext('2d');

3. Inicio la lista de imágenes, índices y coordenadas:

var imgList = new Array();
var wrapper = null;
var spriteIndex = 0;
var spriteX = 0;
var spriteY = 0;

4. Implemento el método 'init' que llamábamos desde el evento de carga de la página. En el ejemplo, creo una precarga para la primera imagen (solución pactada con Raúl Martín, uno de los expertos en javascript de la empresa). Así, en los navegadores comentados se da el mismo comportamiento. Una vez cargada la imagen, deshacemos el nodo creado para la precarga:

init = function() {
  for (var i=0; i<7; ++i)
  {
    imgList.push(new Image());
    imgList[i].src = "http://estaticos.elmundo.es/pruebas/ricky/html5/img/Frame" + i + ".png";
  }

  var img = document.createElement('IMG');
  img.src=  "http://estaticos.elmundo.es/pruebas/ricky/html5/img/Frame0.png";
  img.className = 'precarga';
  img.elemento = imgList[0];

  document.getElementsByTagName('BODY')[0].appendChild(img);

  canvas.width = 1000;
  canvas.height = 130;
  img.onload = function(){
    ctx.fillStyle = "rgba(255,0,0,1)";
    ctx.fillRect(0,0,canvas.width,canvas.height);
    ctx.drawImage(this.elemento,0,0,this.width,this.height);
    this.parentNode.removeChild(this);
  }
  wrapper = document.getElementById("wrapper");
  wrapper.appendChild(canvas);
}

Como podéis ver, desde esta función inicio todo el escenario:

- Relleno la lista de imágenes,

- doy valores al canvas,

- lo dibujo

- y lo añado como hijo del 'wrapper' (sí, suena muy castizo) que creamos en el HTML, previamente.

5. Capturamos los eventos de teclado necesarios. Para que el 'sprite' se mueva, como dije, el usuario ha de pulsar los cursores derecha o izquierda:

document.onkeydown = function (event) {
    var teclaC;

    if ( event == null ) {
        teclaC = window.event.keyCode;
    }
    else { 
        teclaC = event.keyCode;
    }

    switch (teclaC) {
        case 39:animateFrame(1);
            break; 
        case 37:animateFrame(-1);
            break; 
        default: resetFrames();
    }
}

Ahora, cada vez que mantegamos pulsado uno de los dos cursores, habrá por detrás un señor bajito dibujando muy rápidamente -cada cuadro- para mostrarlo después. Esto, en animación, lo llaman intercalador.

6. Vamos a animarlo un poco:

animateFrame = function (dir) {
   if ( dir > 0)
   {
        spriteIndex = ( spriteIndex >= 6 ) ? 0 : ++ spriteIndex;
        spriteX = ( spriteX >= 1000 ) ? 1000 : spriteX + 5;
   }
   else
   {
        spriteIndex = ( spriteIndex <= 0 ) ? 6 : -- spriteIndex;
        spriteX = ( spriteX <= 0 ) ? 0 : spriteX + 5;
   }
   ctx.clearRect(0,0,canvas.width, canvas.height);
 
   ctx.fillStyle = "rgba(255,0,0,1)";
   ctx.fillRect(0,0,canvas.width,canvas.height);
   ctx.drawImage(imgList[spriteIndex],spriteX,spriteY,imgList[spriteIndex].width,imgList[spriteIndex].height);
}

Al igual que en animación tradicional, la sensación de movimiento la generamos mostrando distintos 'frames' en el tiempo. Lo primero que tengo en cuenta es la dirección hacia la que se está moviendo. Si es hacia adelante, sumamos la coordenada 'X'. Si es hacia atrás, restamos.

Utilizo la misma lógica, en este básico caso, para el índice del 'sprite' a mostrar. Es decir, incremento el frame o lo decremento.

Como se puede observar, también, 'repintamos' de nuevo todo con la nueva imagen. Al hacerlo tan rápido, a no ser que estéis aún con un 386, una calculadora científica, un terminal android (es coña, a mi me encantan también) o algo similar, no nos damos ni cuenta.

7. Volviendo al estado inicial:

Cada vez que soltemos el teclado, debemos volver al 'sprite' inicial, para dar coherencia a la animación. Si no realizamos esta acción, el sprite quedaría en la última posición representada. Eso es un error de concepto, así que lo inicializamos de nuevo:

resetFrames = function()
{
   ctx.clearRect(0,0,canvas.width, canvas.height);
   spriteIndex = 0;
  
   ctx.fillStyle = "rgba(255,0,0,1)";
   ctx.fillRect(0,0,canvas.width,canvas.height);
   ctx.drawImage(imgList[spriteIndex],spriteX,spriteY,imgList[spriteIndex].width,imgList[spriteIndex].height);
}

8. Refactorizando:

Como podemos observar, hay 3 líneas que -continuamente- se repiten. Podemos refactorizar el código para que quede mejor:
updateCanvas = function ()

{ 
  ctx.fillStyle = "rgba(255,0,0,1)";
  ctx.fillRect(0,0,canvas.width,canvas.height);
  ctx.drawImage(imgList[spriteIndex],spriteX,spriteY,imgList[spriteIndex].width,imgList[spriteIndex].height);
}

Es decir, donde se encuentren estas tres líneas podemos sustituirlas por una llamada a esta función. Enr esumen: updateCanvas();

NOTA: Recordad que esta última función es posible gracias a que declaramos globales ciertas variables.

9. Hilando fino, fino:

Si queréis, para que nuestro ejemplo quede completo, lo lógico sería que comprobásemos que el navegador de nuestro usuario soporta el elemento Canvas. Para ello, os mando una referencia a Dive into HTML donde existe una documentación muy interesante que habla de esto y de HTML 5 en general.

Aún así, la comprobación podría resumirse en: 

return !!document.createElement('canvas').getContext; // El resultado de la función nos da la respuesta


10. Finalizando:

Resumiendo: con el elemento Canvas podemos controlar lo que mostramos muy dinámicamente, con interacción y de una manera sencilla para realizar operaciones más complejas de representación gráfica.
